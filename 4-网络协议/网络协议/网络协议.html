# 缓存

# 强缓存与协商缓存

# 强缓存

​ 对强缓存来说，浏览器会判断请求的资源是否在有效期内。如果是在有效期内，就直接从缓存中读取资源，不用向服务器发送资源请求。强缓存通过Expires、Cache-Control和Pragma这三个头部字段设置。

Cache-Control：

no-cache: 如果有值，则表明不使用强缓存

no-store: 表示不使用缓存

public：资源可以被浏览器和代理服务器进行缓存

private：资源只能被浏览器进行缓存

# 协商缓存

​ 协商缓存是在用本地缓存之前，会向服务器发起一次GET请求，验证浏览器保存在本地的资源是否过期。

由于 HTTP协议的不足：

  1. 通信使用明文，内容被窃听
  2. 不验证通信双方的身份，可能会遭遇伪装
  3. 无法证明报文的完整性，可能会被篡改

所以需要加密，则为 HTTPS（HTTP+SSL）

# TCP 与 UDP 区别

  1. TCP 面向连接，在通信前需要连接；UDP 是无连接，发送数据前无需连接
  2. TCP 提供可靠的服务，TCP 传送的数据无差错不丢失不重复且按序到达；UDP 尽最大努力交付，不保证可靠交付
  3. TCP 面向字节流，将数据看成一串串无结构的字节流；UDP 是面向报文。
  4. TCP 只能点到点；UDP 可以是一对一，一对多，多对多，多对一
  5. TCP 首部开销 20 字节，UDP 首部开销小，8 字节
  6. TCP 逻辑通信信道是全双工可靠的信道，UDP 是不可靠信道

# DNS

输入：https://www.google.com/ 时，浏览器会通过这个域名去寻找对应的 IP 地址，域名只是方便记忆的别名。

DNS 的作用就是通过域名查询到具体的 IP

# 递归查询：

当你在浏览器想访问 www.google.com 时，会通过一下操作:

1. 本地客户端输入域名向服务器发起请求查询 IP 地址
2. 查看浏览器有没有该域名的 IP 缓存
3. 查看操作系统有没有该域名的 IP 缓存
4. 如果至此还没有命中域名，才会真正的请求本地域名服务器（LDNS）来解析域名，这台服务器一般在城市的某个角落，距离不会很远，并且这条服务器的性能很好，一般都会缓存域名解析的结果，大概 80%的域名解析到这里就完成了
5. 如果这时候还没有的话，会通过直接去 DNS 根服务器查询，这一步查询会找出负责 com 这个一级域名的服务器。
6. 然后去该服务器查询 google.com 这个二级域名
7. 接下来查询 www.google.com 这个三级域名的地址
8. 返回给 LDNS ，并缓存域名和对应的 ip 地址
9. LDNS 把解析的结果返回给客户端并缓存起来
# 迭代查询

区别是前者是由系统配置的 DNS 服务器做请求，得到结果后将数据返回给客户端，后者是由客户端去做请求。

# CDN

cdn 全称content delivery network ，内容分发网络。CDN 是构建在网络基础上的智能虚拟网络，依靠部署在各地的边缘服务器，通过中心平台的负载均衡，内容分发，调度等功能模块，使用户就近获取所需内容，减低网络阻塞，提高用户访问响应速度和命中率。

CDN 是一个经策略性部署的整体系统，包括分布式存储，负载均衡，网络请求的重定向和内容管理，其中内容管理和全局的网络流量管理是CDN的核心。
